基調講演: 日本Pythonコミュニティの１０年ちょっと - 日本におけるPython普及の歴史と展望

石本 敦夫(アクシスソフト(株) シニアアーキテクト)
atsuoishimotoの日記(http://d.hatena.ne.jp/atsuoishimoto/)

石本氏は日本でも屈指のPythonアーリーアダプタで、python-ml-jp メーリングリストやPythonユーザ会のウェブサイトpython.jpの立ち上げ・運営に尽力されています。

古くはPythonの日本語文字コード変換ライブラリpykfを公開され、最近ではPEP 3138を提案・実装されるなど、日本のPythonユーザをとりまくプログラミング環境の改善と世界への橋渡しに大きな功績を残されています。最近でも、コミュニケーションツールInfoPileのリリースやPython Hack-a-thonの講師を勤められるなど、精力的に活動されています。

石本氏の深い洞察と研鑽の歴史から紡ぎだされるPython語りを、ぜひ聴きに来てください!

Python 界隈の翻訳プロジェクト

森本 哲也(株式会社データ変換研究所)
forest book(http://d.hatena.ne.jp/t2y-1979/)

Python 界隈の翻訳プロジェクトを紹介します。例えば、以下のような翻訳ドキュメントがあります。
  • Python 標準ドキュメント (http://www.python.jp/doc/2.6/)
  • Python Module Of The Week (http://www.doughellmann.com/PyMOTW-ja/)
  • Zope 2 関連ドキュメント (http://docs.zope.jp/zope2/)
  • SQLAlchemy 0.6.5 ドキュメント (http://omake.accense.com/static/doc-ja/sqlalchemy/)
  • Sphinx ドキュメント (http://sphinx-users.jp/doc.html)
Python のことを学ぶにあたって和訳されたドキュメントはまだまだ少ないです。また翻訳に携わっているメンバーも少ないため、最新バージョンのドキュメントの翻訳が追いつかない状況もあったりします。翻訳ってどうやって行っているのかな？という疑問への回答、翻訳ちょっとやってみようかな？という気になるように翻訳プロジェクトへの参加方法、便利なツールや翻訳を継続するためのノウハウ等を共有します。

Sphinx 1.1 のi18n機能紹介

Ian Lewis(BeProud)
Ian Lewie(http://www.ianlewis.org/)

Sphinx は Restructured Text を利用しているドキュメントビルドシステムです。
Sphinx 1.0までは、国際化したドキュメントは完全に別で管理し、元のドキュメントが変更された時に翻訳ドキュメントに何を修正すればいいか分からなくて、大変でした。
Sphinx 1.1 では、i18n機能を導入し、ドキュメントの翻訳がもっと簡単にできるようになります。
この発表では、Sphinx の i18n 機能をどうやって使えばいいかを説明します。
